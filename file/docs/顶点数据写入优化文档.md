# std::vector 顶点数据写入性能对比

## 三种方案对比

### 方案1: push_back（原始方案）

```cpp
// 逐个 push_back，可能触发多次扩容
for (const auto& pos : positionIterable) {
    primitiveData._positions.push_back(pos.x());
    primitiveData._positions.push_back(pos.y());
    primitiveData._positions.push_back(pos.z());
}
```

**问题**:
- 容量不足时触发扩容: 分配新内存 → 拷贝旧数据 → 释放旧内存
- 时间复杂度: O(n²)
- 频繁的内存分配和释放

---

### 方案2: resize + operator[]（改进方案）

```cpp
// 预分配内存
primitiveData._positions.resize(vertexCount * 3);

// 通过索引写入
for (size_t i = 0; i < vertexCount; ++i) {
    primitiveData._positions[i * 3 + 0] = position[i].x();
    primitiveData._positions[i * 3 + 1] = position[i].y();
    primitiveData._positions[i * 3 + 2] = position[i].z();
}
```

**优点**:
- 只分配一次内存
- 无扩容开销

**缺点**:
- 需要计算索引 `i * 3`，有乘法开销
- `operator[]` 在 debug 模式可能有边界检查

---

### 方案3: resize + 指针递增（最优方案）

```cpp
// 预分配内存
primitiveData._positions.resize(vertexCount * 3);

// 指针递增写入
float* ptr = primitiveData._positions.data();
for (const auto& pos : positionIterable) {
    *ptr++ = pos.x();
    *ptr++ = pos.y();
    *ptr++ = pos.z();
}
```

**优点**:
- 只分配一次内存
- 无索引计算（递增比乘法快）
- 编译器容易优化为 SIMD
- 代码简洁

---

## 性能对比表

| 方案 | 内存分配 | 索引计算 | 扩容开销 | 推荐 |
|------|---------|---------|---------|------|
| push_back | 多次 | 无 | 高 | ❌ 不推荐 |
| resize + [] | 1次 | 有（乘法） | 无 | ⭕ 可用 |
| resize + 指针 | 1次 | 无 | 无 | ✅ 推荐 |

## 进一步优化：直接生成 Interleaved 数据

### 当前问题（两次遍历）

```
fastgltf accessor → _positions/_normals/_uvs... (分开存储)
                ↓
                MeshAsset::_prepareVertexData 再次遍历，interleave
                ↓
                上传GPU
```

### 优化方案（一次遍历）

```cpp
// 计算顶点步长和总大小
size_t vertexStride = 3 + 3 + 2 + 2 + 2 + 4 + 4; // 20 floats
size_t vertexCount = positionAccessor.count;

// 预分配 interleaved 数据
primitiveData._interleavedVertices.resize(vertexCount * vertexStride);

// 一次遍历，直接写入 interleaved 格式
float* ptr = primitiveData._interleavedVertices.data();

for (size_t i = 0; i < vertexCount; ++i) {
    // Position (3 floats)
    *ptr++ = position[i].x();
    *ptr++ = position[i].y();
    *ptr++ = position[i].z();

    // Normal (3 floats) - 存在则写入，否则写默认值
    if (hasNormal) {
        *ptr++ = normal[i].x();
        *ptr++ = normal[i].y();
        *ptr++ = normal[i].z();
    } else {
        *ptr++ = 0.0f; *ptr++ = 0.0f; *ptr++ = 1.0f;
    }

    // UV0 (2 floats)
    if (hasUV0) {
        *ptr++ = uv0[i].x();
        *ptr++ = uv0[i].y();
    } else {
        *ptr++ = 0.0f; *ptr++ = 0.0f;
    }

    // ... 其他属性
}
```

**收益**:
- 减少一次完整的数据遍历
- 减少内存占用（不需要分开存储）
- MeshAsset 可直接上传，无需二次处理

## 索引数据优化

### 当前方案（统一转 uint32_t）

```cpp
switch (idxAccessor.componentType) {
    case fastgltf::ComponentType::UnsignedByte:
        for (auto index : indices) primitiveData._indices.push_back(index);
        break;
    case fastgltf::ComponentType::UnsignedShort:
        for (auto index : indices) primitiveData._indices.push_back(index);
        break;
    // ...
}
```

**问题**: 所有索引都转 uint32_t，浪费内存

### 优化方案（保持原始类型）

```cpp
// 根据顶点数量决定索引类型
if (vertexCount <= 65535) {
    primitiveData._indexType = IndexType::UInt16;
    primitiveData._indices16.resize(indexCount);
    // 拷贝 uint16_t 数据
} else {
    primitiveData._indexType = IndexType::UInt32;
    primitiveData._indices32.resize(indexCount);
    // 拷贝 uint32_t 数据
}
```

**收益**:
- 顶点 < 65535 时，索引内存减少 50%
- 更好的 CPU cache 命中率

---

## 总结

| 优化项 | 原始做法 | 优化做法 | 预期收益 |
|--------|---------|---------|---------|
| 顶点写入 | push_back | resize + 指针 | 2-5x |
| 数据布局 | 分开存储再interleave | 直接interleave | 2-3x |
| 索引类型 | 统一uint32_t | 按需uint16/32 | 内存-50% |
| 传参方式 | 值拷贝 | 移动语义 | 10-20% |
