## 1. 什么是投影矩阵

### 1.1 定义

**投影矩阵（Projection Matrix）** 是 3D 图形渲染中的核心矩阵之一，它的作用是：

> 将**摄像机坐标空间**中的 3D 坐标转换为**NDC（归一化设备坐标）** 或 **屏幕坐标**

简单来说，投影矩阵回答的问题是：
- 使用什么镜头？（广角/长焦）
- 视野范围有多大？
- 近平面和远平面在哪里？

### 1.2 在渲染管线中的位置

```
世界坐标 → 视图矩阵 → 摄像机坐标 → 投影矩阵 → NDC → 屏幕坐标
```

具体例子：

| 阶段 | 坐标 | 说明 |
|------|------|------|
| 摄像机坐标 | `(5, 3, -10)` | 相对于摄像机前方 10 米 |
| ↓ 投影矩阵 | | |
| NDC | `(0.2, 0.15, 0.7)` | 归一化设备坐标 (-1~1 或 0~1) |
| ↓ 视口变换 | | |
| 屏幕坐标 | `(400, 300)` | 像素坐标 |

### 1.3 两种投影方式

| 投影方式 | 特点 | 用途 |
|----------|------|------|
| **透视投影** | 近大远小 | 3D 游戏、模拟真实场景 |
| **正交投影** | 远近一样大 | UI、2D 游戏、工程制图 |

## 2. 透视投影原理

### 2.1 什么是透视投影

透视投影模拟了人眼的视觉效果：**近处的物体大，远处的物体小**。

```
          摄像机
            │
            │\
            │ \
            │  \
            │   \  远处的物体（看起来小）
            │    \
            │     \
            │      \  近处的物体（看起来大）
            │       \
            └────────\
                  屏幕
```

### 2.2 核心参数

透视投影由四个参数定义：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| **FOV** | 视野角度（垂直方向） | 45° ~ 90° |
| **Aspect** | 宽高比 | 16:9, 4:3 |
| **Near** | 近裁剪面 | 0.1, 1.0 |
| **Far** | 远裁剪面 | 100, 1000, 2000 |

### 2.3 视锥体

透视投影形成一个**视锥体（Frustum）**：

```
           /\
          /  \
         /    \
        /      \
       /   [Far] \
      /___________\
     \           /
      \   [Near]/
       \________/
          摄像机
```

- **Near 平面**：近处的裁剪面，在此之前的物体不渲染
- **Far 平面**：远处的裁剪面，在此之后的物体不渲染

## 3. 透视投影矩阵公式推导

### 3.1 目标

将摄像机空间中的点 `(x, y, z)` 变换到 NDC 空间。

### 3.2 透视除法

透视投影的核心是**透视除法**：

```cpp
// 原始坐标
float x_view, y_view, z_view;

// 透视除法（w 分量）
float x_ndc = x_view / (-z_view);  // -z 因为摄像机看向 -Z 方向
float y_ndc = y_view / (-z_view);
```

这解释了为什么**近处的物体大**（分母小，结果大）和**远处的物体小**（分母大，结果小）。

### 3.3 推导过程

#### 步骤1：建立相似三角形

```
                y
                │
                │    top
         ───────┼───────  near
               /│\
              / │ \
             /  │  \
            /   │   \
           /    │    \
          /     │     \
   ──────/──────┼──────\──────  far
      -right   /│\    left
              z│ │
               │ │
               ▼ ▼
              -z
```

根据相似三角形：

```
y_ndc / (-z) = top / (-near)
y_ndc = top * (-z) / (-near) = top * z / near
```

#### 步骤2：归一化到 [-1, 1]

```cpp
// 将 [-top, top] 映射到 [-1, 1]
y_ndc = (y_view / top) * (-near / -z_view)
       = y_view / (top * (-z_view) / near)

// 同样，x_ndc
x_ndc = (x_view / (right * (-z_view) / near))
```

#### 步骤3：引入宽高比

```cpp
// aspect = width / height
// right = top * aspect

x_ndc = x_view / (aspect * top * (-z_view) / near)
      = x_view * near / (aspect * top * (-z_view))

y_ndc = y_view / (top * (-z_view) / near)
      = y_view * near / (top * (-z_view))
```

### 3.4 透视投影矩阵

```
┌────────────────────────────────────────────────────────────┐
│  1/(aspect*tan(fov/2))        0              0          0  │
│         0              1/tan(fov/2)         0          0  │
│         0                    0      -(far+near)/range  -1  │
│         0                    0    -2*far*near/range    0  │
└────────────────────────────────────────────────────────────┘

其中：range = far - near
```

### 3.5 矩阵元素详解

| 元素 | 公式 | 作用 |
|------|------|------|
| `m[0][0]` | `1 / (aspect * tanHalfFov)` | X 轴缩放 |
| `m[1][1]` | `1 / tanHalfFov` | Y 轴缩放 |
| `m[2][2]` | `-(far + near) / range` | Z 轴深度映射 |
| `m[2][3]` | `-1.0f` | 透视除法标志 |
| `m[3][2]` | `-2 * far * near / range` | Z 轴平移 |

## 4. 正交投影原理

### 4.1 什么是正交投影

正交投影没有透视效果：**近处和远处的物体一样大**。

```
    ┌─────────────────┐
    │                 │
    │    远处的方块    │────┐
    │                 │    │
    ├─────────────────┤    │  一样大！
    │                 │    │
    │    近处的方块    │────┘
    └─────────────────┘
```

### 4.2 核心参数

正交投影由六个参数定义：

| 参数 | 说明 |
|------|------|
| **Left** | 视口左边界 |
| **Right** | 视口右边界 |
| **Bottom** | 视口下边界 |
| **Top** | 视口上边界 |
| **Near** | 近裁剪面 |
| **Far** | 远裁剪面 |

## 5. 正交投影矩阵公式推导

### 5.1 推导过程

正交投影更简单，只需要线性映射：

```cpp
// X 轴： [left, right] → [-1, 1]
x_ndc = (x_view - left) / (right - left) * 2 - 1
      = 2 * x_view / (right - left) - (right + left) / (right - left)

// 简化形式
x_ndc = x_view * (2 / (right - left)) - (right + left) / (right - left)
```

### 5.2 正交投影矩阵

```
┌────────────────────────────────────────────────────────────┐
│       2/(right-left)              0              0       0  │
│              0             2/(top-bottom)        0       0  │
│              0                    0        -2/(far-near)  0  │
│  -(right+left)/(right-left)  -(top+bottom)/(top-bottom) -(far+near)/(far-near)  1  │
└────────────────────────────────────────────────────────────┘
```

## 6. Boo Engine 实际实现

### 6.1 透视投影实现

Boo Engine 中的 `Mat4::perspective` 函数实现：

```cpp
void Mat4::perspective(Mat4 &out, float fov, float aspect, float near, float far, float projectionSignY, int orientation)
{
    float tanHalfFov = tan(fov * 0.5f);
    // 计算矩阵元素
    float m00 = 1.0f / (aspect * tanHalfFov);
    float m11 = 1.0f / tanHalfFov;
    float m22 = far / (near - far);
    float m23 = -1.0f; // 注意：这是用于生成深度值
    float m32 = -(far * near) / (far - near);

    out._m[0] = m00;
    out._m[1] = 0.0f;
    out._m[2] = 0.0f;
    out._m[3] = 0.0f;
    out._m[4] = 0.0f;
    out._m[5] = m11 * projectionSignY;
    out._m[6] = 0;
    out._m[7] = 0;
    out._m[8] = 0;
    out._m[9] = 0;
    out._m[10] = m22;
    out._m[11] = m23;
    out._m[12] = 0.0f;
    out._m[13] = 0.0f;
    out._m[14] = m32;
    out._m[15] = 0.0f;
}
```

### 6.2 正交投影实现

Boo Engine 中的 `Mat4::ortho` 函数实现：

```cpp
void Mat4::ortho(Mat4 &out, float left, float right, float bottom, float top, float near, float far, float projectionSignY, int orientation)
{
    // OpenGL 向上 1   Vulkan 向下 -1  Metal 向下 -1  * projectionSignY
    float lr = 1.0f / (right - left);
    float bt = 1.0f / (top - bottom);
    float nf = 1.0f / (far - near);

    float x = 2.0f * lr;
    float y = 2.0f * bt;
    float dx = -(right + left) * lr;
    float dy = -(top + bottom) * bt;

    out._m[0] = x;
    out._m[1] = 0;
    out._m[2] = 0;
    out._m[3] = 0;
    out._m[4] = 0;
    out._m[5] = y * projectionSignY;
    out._m[6] = 0;
    out._m[7] = 0;
    out._m[8] = 0;
    out._m[9] = 0;
    out._m[10] = nf;
    out._m[11] = 0;
    out._m[12] = dx;
    out._m[13] = dy;
    out._m[14] = -near * nf;
    out._m[15] = 1;
}
```

### 6.3 矩阵形式对比

**透视投影矩阵**（列主序存储）：
```
// 列0
[ 1/(aspect*tanHalfFov), 0, 0, 0 ]

// 列1
[ 0, 1/tanHalfFov * projectionSignY, 0, 0 ]

// 列2
[ 0, 0, far/(near-far), -1 ]

// 列3
[ 0, 0, -(far*near)/(far-near), 0 ]

// 实际存储顺序：
// _m[0] = 1/(aspect*tanHalfFov), _m[1] = 0, _m[2] = 0, _m[3] = 0
// _m[4] = 0, _m[5] = 1/tanHalfFov * projectionSignY, _m[6] = 0, _m[7] = 0
// _m[8] = 0, _m[9] = 0, _m[10] = far/(near-far), _m[11] = -1
// _m[12] = 0, _m[13] = 0, _m[14] = -(far*near)/(far-near), _m[15] = 0
```

**正交投影矩阵**（列主序存储）：
```
// 列0
[ 2/(right-left), 0, 0, 0 ]

// 列1
[ 0, 2/(top-bottom) * projectionSignY, 0, 0 ]

// 列2
[ 0, 0, 1/(far-near), 0 ]

// 列3
[ -(right+left)/(right-left), -(top+bottom)/(top-bottom), -near/(far-near), 1 ]

// 实际存储顺序：
// _m[0] = 2/(right-left), _m[1] = 0, _m[2] = 0, _m[3] = 0
// _m[4] = 0, _m[5] = 2/(top-bottom) * projectionSignY, _m[6] = 0, _m[7] = 0
// _m[8] = 0, _m[9] = 0, _m[10] = 1/(far-near), _m[11] = 0
// _m[12] = -(right+left)/(right-left), _m[13] = -(top+bottom)/(top-bottom), _m[14] = -near/(far-near), _m[15] = 1
```

## 7. 坐标系差异

### 7.1 不同图形 API 的 NDC 范围

| API | NDC Z 范围 | 摄像机看向 |
|-----|------------|-----------|
| **OpenGL** | [-1, 1] | -Z |
| **Vulkan** | [0, 1] | -Z |
| **DirectX** | [0, 1] | +Z |

### 7.2 Boo Engine 使用

Boo Engine 基于 **Vulkan**，NDC 深度范围是 **[0, 1]**。

这意味着：
- 近平面（Near）→ NDC Z = 0
- 远平面（Far）→ NDC Z = 1

### 7.3 公式差异

| API | m[2][2] | m[3][2] |
|-----|---------|---------|
| OpenGL | -(f+n)/(f-n) | -2fn/(f-n) |
| Vulkan | -(f+n)/(f-n) | -2fn/(f-n) |

两者公式相同，因为 Vulkan 和 OpenGL 的投影矩阵公式实际上是一样的，只是后续处理有所不同。

## 8. 常见问题

### 8.1 物体不显示

| 原因 | 解决方案 |
|------|----------|
| 物体在 Near 平面之前 | 增大 Near 值或移动物体 |
| 物体在 Far 平面之后 | 增大 Far 值或移动物体 |
| 宽高比错误 | 检查 aspectRatio 计算 |

### 8.2 画面变形

| 原因 | 解决方案 |
|------|----------|
| 宽高比不正确 | 检查 `view->getWidth() / view->getHeight()` |
| FOV 过大 | 减小 FOV 值（推荐 45°~60°） |

### 8.3 深度缓冲问题

| 问题 | 原因 |
|------|------|
| 深度冲突（Z-fighting） | Near 和 Far 差距过大 |
| 精度不足 | 增大 Far 或减小 Near |

### 8.4 tanHalfFov 的由来

```cpp
float tanHalfFov = tan(fov * 3.14159f / 360.0f);
```

- `fov * 3.14159f / 360.0f` = 将角度转换为弧度（fov/2）
- `tan(fov/2)` = fov 一半的正切值

几何意义：
```
       |\
       | \
       |  \  fov/2
       |   \
       |____\
      tan(fov/2) = opposite / adjacent
```

## 总结

1. **透视投影**：模拟人眼，近大远小
   - 核心：透视除法（除以 -z）
   - 参数：FOV、Aspect、Near、Far

2. **正交投影**：无透视效果，远处和近处一样大
   - 核心：线性映射
   - 参数：Left、Right、Top、Bottom、Near、Far

3. **投影矩阵的作用**：
   - 将 3D 坐标转换为 NDC
   - 决定可见范围（视锥体裁剪）
   - 产生透视效果（透视投影）
