# FreeType 字体渲染系统设计文档

## 概述

FreetypeMgr 是 Boo 引擎中的字体渲染管理器，基于 FreeType 库实现高效的文本渲染。其核心设计思想是将字形（Glyph）预先渲染到纹理图集（Texture Atlas）中，渲染时通过 UV 坐标从图集中采样获取字形图像，避免了每次渲染文本时都进行字形栅格化的高昂开销。

### 核心特性

- **字形图集（Glyph Atlas）**：将多个字形打包到单张纹理中，减少 Draw Call
- **按需缓存（Lazy Caching）**：仅在需要时加载和缓存字形，支持动态文本
- **多图集支持**：当一个图集页满时自动创建新页面
- **UTF-8 完整支持**：通过 `std::wstring` 转换支持中文等多字节字符
- **网格批处理（Mesh Batching）**：同一字图集的字符共享批次，优化渲染性能

## 核心数据结构

### GlyphData - 字形数据

```cpp
struct GlyphData
{
    wchar_t character;              // 字符的 Unicode 码点
    uint32_t width;                 // 字形位图宽度（像素）
    uint32_t height;                // 字形位图高度（像素）
    int32_t offsetX;                // 在图集中的 X 偏移
    int32_t offsetY;                // 在图集中的 Y 偏移
    std::vector<uint8_t> bitmapData; // 字形位图数据（灰度）
};
```

**设计说明**：
- `bitmapData` 存储灰度图像，渲染时通过 `R8G8B8A8_SRGB` 格式转换为 RGBA
- `offsetX/Y` 记录字形在图集中的位置，用于计算 UV 坐标

### FontAtlas - 字体图集页

```cpp
struct FontAtlas
{
    std::string uuid;                    // 图集唯一标识
    TextureAsset *texture = nullptr;     // 对应的 GPU 纹理
    size_t width = 0;                    // 图集宽度（默认 2048）
    size_t height = 0;                   // 图集高度（默认 2048）
    size_t cursorX = 0;                  // 当前写入位置 X
    size_t cursorY = 0;                  // 当前写入位置 Y
    bool isFull = false;                 // 是否已满
    bool needsUpdate = false;            // 是否需要更新 GPU 数据
    std::map<wchar_t, GlyphData> glyphs; // 该页所有字形
};
```

**设计说明**：
- 使用**行打包算法**（Row Packing）：从左到右写入，超出宽度则换行
- `needsUpdate` 标记用于批量更新 GPU 纹理，减少 CPU-GPU 数据传输
- 多图集设计：当前一页写满时，自动创建新图集页

### TextMeshBatch - 文本网格批次

```cpp
struct TextMeshBatch
{
    std::string fontAtlasUuid;           // 关联的图集 UUID
    TextureAsset *texture = nullptr;     // 使用的纹理
    std::vector<float> positions;        // 顶点坐标 (x, y, z)
    std::vector<float> uvs;              // UV 坐标 (u, v)
    std::vector<int> indices;             // 索引数据
};
```

**设计说明**：
- 相同图集的字符合并到同一批次，减少 Draw Call
- 使用 `positions` 和 `uvs` 双数组存储，顶点按顺序对应

### TextLayoutResult - 文本布局结果

```cpp
struct TextLayoutResult
{
    int fontSize = 50;                   // 字体大小
    uint32_t width = 0;                  // 文本总宽度
    uint32_t height = 0;                // 文本总高度
    std::vector<TextMeshBatch> batches; // 所有纹理批次
};
```

**设计说明**：
- 包含完整的渲染所需数据：网格 + 纹理 + 尺寸信息

## 渲染流程详解

### 整体流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  1. init()   │───▶│ 2. create()  │───▶│ 3. 文本解析   │───▶│ 4. 网格生成  │
│  初始化 FreeType │    │  外部调用入口 │    │  字形缓存     │    │  批次组织    │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

### 步骤 1：初始化 (init)

```cpp
void FreetypeMgr::init()
{
    // 1. 设置字体参数
    this->_fontSpace = 4;              // 字间距
    this->_fontSize = 50;              // 默认字体大小
    this->_fontBaseLine = 50 * 0.95;   // 基准线位置
    this->_fontLineHeight = 50 * 1.1;  // 行高

    // 2. 创建初始图集页
    this->_createFontAtlas();

    // 3. 初始化 FreeType 库
    FT_Init_FreeType(&this->_ft);

    // 4. 加载字体文件
    // Windows: C:\Windows\Fonts\simhei.ttf
    // macOS: /System/Library/Fonts/STHeiti Light.ttc
    FT_New_Face(this->_ft, this->_fontPath.c_str(), 0, &this->_face);

    // 5. 预热缓存：加载常用字符
    this->_parseText("ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz...");
}
```

**设计意图**：
- 预加载常用字符到图集，避免运行时首次渲染卡顿
- 使用系统自带字体，无需打包额外资源

### 步骤 2：外部调用 (create)

```cpp
TextLayoutResult FreetypeMgr::create(const std::string &text, int fontSize)
{
    // 1. 解析文本，缓存缺失的字形到图集
    this->_parseText(text);

    // 2. 计算文本尺寸
    TextLayoutResult result;
    std::vector<std::string> lines = this->_splitTextLines(text);
    this->_getTextContentSize(lines, result.width, result.height);

    // 3. 生成网格数据
    this->_layoutText(lines, result);

    // 4. 根据目标字体大小进行缩放
    float rate = 1.0f * fontSize / this->_fontSize;
    result.fontSize = fontSize;
    result.width *= rate;
    result.height *= rate;

    return result;
}
```

### 步骤 3：文本解析与字形缓存 (_parseText)

```cpp
void FreetypeMgr::_parseText(const std::string &text)
{
    // 1. UTF-8 转宽字符
    std::wstring wstr = FmtUtil::utf8_to_wstring(text);

    // 2. 遍历每个字符
    for (wchar_t c : wstr)
    {
        // 跳过空白字符
        if (std::iswspace(c)) continue;

        // 检查是否已缓存
        if (this->_isGlyphCached(c)) continue;

        // 3. 加载字形位图
        GlyphData glyphData{};
        glyphData.character = c;
        this->_loadGlyph(glyphData);

        // 4. 写入图集
        this->_writeToAtlas(glyphData);
    }

    // 5. 批量更新 GPU 纹理
    for (FontAtlas &atlas : this->_atlasPages)
    {
        if (atlas.needsUpdate)
        {
            GfxTexture *texture = atlas.texture->getGfxTexture();
            texture->updateData();
            atlas.needsUpdate = false;
        }
    }
}
```

**关键设计**：
- **按需加载**：仅缓存缺失的字形
- **批量更新**：所有图集页一次性上传 GPU，减少开销
- **灰度转 RGBA**：在写入纹理时将灰度值扩展为 RGBA

### 步骤 4：字形加载 (_loadGlyph)

```cpp
void FreetypeMgr::_loadGlyph(GlyphData &glyphData)
{
    // 1. 设置字体大小
    FT_Set_Pixel_Sizes(this->_face, 0, this->_fontSize);

    // 2. 加载字形（同时进行栅格化）
    FT_Load_Char(this->_face, glyphData.character, FT_LOAD_RENDER);

    FT_GlyphSlot glyph = this->_face->glyph;
    uint32_t width = glyph->bitmap.width;
    uint32_t height = glyph->bitmap.rows;

    // 3. 计算字形参数
    uint32_t bearingX = glyph->bitmap_left;    // 水平偏移
    uint32_t bearingY = glyph->bitmap_top;     // 垂直偏移
    uint32_t advance = glyph->advance.x >> 6;  // 推进距离

    // 4. 偏移计算：调整位图位置使基线对齐
    int offsetY = _fontBaseLine - bearingY;

    // 5. 创建位图数据（扩展到完整行高）
    std::vector<uint8_t> datas(width * this->_fontLineHeight, 0);
    for (uint32_t y = 0; y < height; y++)
    {
        for (uint32_t x = 0; x < width; x++)
        {
            uint8_t gray = glyph->bitmap.buffer[y * width + x];
            datas[(y + offsetY) * width + x] = gray;  // Y 轴偏移
        }
    }

    glyphData.width = width;
    glyphData.height = this->_fontLineHeight;
    glyphData.bitmapData = datas;
}
```

**关键技术点**：
- **位图坐标转换**：FreeType 的位图原点在左下角，需要转换到左上角
- **基线对齐**：通过 `offsetY` 调整，确保同一行字符基线对齐
- **固定行高**：位图高度扩展到 `_fontLineHeight`，保证行间距一致

### 步骤 5：图集写入 (_writeToAtlas)

```cpp
void FreetypeMgr::_writeToAtlas(GlyphData &glyphData)
{
    uint32_t offset_x = 4;  // 像素间隔，防止字形粘连

    // 1. 获取当前图集页
    FontAtlas &atlas = this->_atlasPages.back();

    // 2. 检查是否需要换行
    if (atlas.cursorX + glyphData.width + offset_x > atlas.width)
    {
        atlas.cursorX = 0;
        atlas.cursorY += this->_fontLineHeight;
    }

    // 3. 检查是否需要新建图集页
    if (atlas.cursorY + glyphData.height > atlas.height)
    {
        atlas.isFull = true;
        this->_createFontAtlas();  // 自动创建新图集
        atlas = this->_atlasPages.back();
    }

    atlas.needsUpdate = true;

    // 4. 写入像素数据到纹理
    GfxTexture *texture = atlas.texture->getGfxTexture();
    uint32_t startX = atlas.cursorX + offset_x;
    uint32_t startY = atlas.cursorY;

    glyphData.offsetX = startX;
    glyphData.offsetY = startY;

    for (uint32_t y = 0; y < glyphData.height; y++)
    {
        for (uint32_t x = 0; x < glyphData.width; x++)
        {
            uint8_t pixel = glyphData.bitmapData[y * glyphData.width + x];
            texture->setPixel(startX + x, startY + y, {pixel, pixel, pixel, pixel});
        }
    }

    // 5. 更新游标位置
    atlas.cursorX = startX + glyphData.width;

    // 6. 缓存字形数据
    atlas.glyphs[glyphData.character] = glyphData;
    this->_glyphCache[glyphData.character] = glyphData;
}
```

**图集打包算法**：

```
┌────────────────────────────────────────┐
│  cursorX →                            │
│  cursorY                              │
│  ↓                                    │
│  ┌─────┐ ┌─────┐ ┌─────┐              │
│  │ A   │ │ B   │ │ C   │  ← 换行       │
│  └─────┘ └─────┘ └─────┘              │
│       ┌─────────┐                      │
│       │   D     │                      │
│       └─────────┘                      │
│                                        │
└────────────────────────────────────────┘
```

### 步骤 6：网格布局生成 (_layoutLineText)

```cpp
void FreetypeMgr::_layoutLineText(const std::string &text, float offsetY, ...)
{
    // 1. 转换为宽字符
    std::wstring wstr = FmtUtil::utf8_to_wstring(text);

    // 2. 计算行宽（累加字形宽度 + 字间距）
    float lineWidth = 0;
    for (wchar_t c : wstr)
    {
        if (!this->_isGlyphCached(c)) continue;
        GlyphData &glyphData = this->_glyphCache[c];
        lineWidth += glyphData.width + this->_fontSpace;
    }

    // 3. 计算顶点位置（居中对齐）
    float unit_x = 1.0f / result.width;
    float unit_y = 1.0f / result.height;
    float left = -0.5f + (result.width - lineWidth) / 2.0f * unit_x;
    float top = offsetY * unit_y;
    float bottom = (offsetY - this->_fontLineHeight) * unit_y;

    // 4. 为每个字形生成两个三角形（6个顶点）
    for (wchar_t c : wstr)
    {
        // 查找字形所在图集和批次
        FontAtlas *atlas = this->_findGlyphAtlasPage(c);
        TextMeshBatch *batch = this->_findTextMeshBatch(atlas, result.batches);

        GlyphData &glyphData = atlas->glyphs[c];

        // 计算 UV 坐标
        float uv_x = 1.0f * glyphData.offsetX / atlas->width;
        float uv_y = 1.0f * glyphData.offsetY / atlas->height;
        float uv_w = 1.0f * glyphData.width / atlas->width;
        float uv_h = 1.0f * glyphData.height / atlas->height;

        // 第一个三角形：左上、左下、右下
        batch->positions.push_back(left); batch->positions.push_back(top); batch->positions.push_back(0.0f);
        batch->uvs.push_back(uv_x); batch->uvs.push_back(uv_y);

        batch->positions.push_back(left); batch->positions.push_back(bottom); batch->positions.push_back(0.0f);
        batch->uvs.push_back(uv_x); batch->uvs.push_back(uv_y + uv_h);

        batch->positions.push_back(right); batch->positions.push_back(bottom); batch->positions.push_back(0.0f);
        batch->uvs.push_back(uv_x + uv_w); batch->uvs.push_back(uv_y + uv_h);

        // 第二个三角形：左上、右下、右上
        batch->positions.push_back(left); batch->positions.push_back(top); batch->positions.push_back(0.0f);
        batch->uvs.push_back(uv_x); batch->uvs.push_back(uv_y);

        batch->positions.push_back(right); batch->positions.push_back(bottom); batch->positions.push_back(0.0f);
        batch->uvs.push_back(uv_x + uv_w); batch->uvs.push_back(uv_y + uv_h);

        batch->positions.push_back(right); batch->positions.push_back(top); batch->positions.push_back(0.0f);
        batch->uvs.push_back(uv_x + uv_w); batch->uvs.push_back(uv_y);

        // 添加索引
        int idx = batch->positions.size() / 3 - 6;
        batch->indices.push_back(idx, idx+1, idx+2, idx+3, idx+4, idx+5);
    }
}
```

### UIText 组件

`UIText` 是暴露给上层的文本组件，其渲染流程如下：

```cpp
void UIText::setText(std::string text)
{
    this->_text = text;

    // 调用 FreetypeMgr 创建文本布局
    TextLayoutResult result = FreetypeMgr::getInstance()->create(text, 50);

    // 更新 Mesh 数据
    this->_realWidth = result.width;
    this->_realHeight = result.height;
    MeshPrimitive primitive;
    primitive._positions = result.batches[0].positions;
    primitive._uvs = result.batches[0].uvs;
    primitive._indices = result.batches[0].indices;
    this->_meshAsset->updateSubMesh(0, primitive);

    // 更新材质纹理
    this->_materialAsset->setTexture(result.batches[0].texture);

    // 更新节点尺寸
    this->_updateNodeSize();
}
```

### UTF-8 到宽字符转换

```cpp
std::wstring FmtUtil::utf8_to_wstring(const std::string &str)
{
    std::wstring_convert<std::codecvt_utf8<wchar_t>> converter;
    return converter.from_bytes(str);
}
```

### 字形缓存查找

```cpp
bool FreetypeMgr::_isGlyphCached(wchar_t character)
{
    return this->_glyphCache.find(character) != this->_glyphCache.end();
}
```

### 多图集批次查找

```cpp
TextMeshBatch *FreetypeMgr::_findTextMeshBatch(FontAtlas *atlas, std::vector<TextMeshBatch> &batches)
{
    // 查找已存在的批次
    for (TextMeshBatch &batch : batches)
    {
        if (batch.fontAtlasUuid == atlas->uuid)
        {
            return &batch;
        }
    }

    // 不存在则创建新批次
    TextMeshBatch batch;
    batch.fontAtlasUuid = atlas->uuid;
    batch.texture = atlas->texture;
    batches.push_back(batch);
    return &batches.back();
}
```

## 性能优化

### 已实现的优化

| 优化项 | 说明 |
|--------|------|
| **字形缓存** | 已渲染的字形不再重复处理 |
| **批量 GPU 更新** | 多个图集页合并为一次 `updateData()` 调用 |
| **网格批处理** | 相同图集的字符共享 `TextMeshBatch` |
| **预热加载** | 初始化时预加载常用 ASCII 字符 |
| **多图集支持** | 单图集满时自动扩展，避免空间浪费 |

### 潜在优化方向

1. **图集打包算法优化**：当前使用简单行打包，可考虑**最佳矩形打包**（Best Rectangle Packing）算法提高空间利用率
2. **Mipmap 支持**：为字体纹理生成 Mipmap，改善远距离文字清晰度
3. ** SDF 字体**：使用 Signed Distance Field 实现任意大小缩放不失真
4. **字形顶点缓存**：对频繁使用的文本进行顶点数据缓存，避免重复计算

## 使用示例

### C++ 调用

```cpp
// 获取字体管理器单例
FreetypeMgr *mgr = FreetypeMgr::getInstance();

// 初始化（通常在引擎启动时调用一次）
mgr->init();

// 创建文本布局
std::string text = "你好，世界！Hello World!";
TextLayoutResult result = mgr->create(text, 32);

// result 包含：
// - result.width: 文本宽度
// - result.height: 文本高度
// - result.batches: 可直接用于渲染的网格批次
```

## 总结

Boo 引擎的字体渲染系统采用了游戏行业经典的 **Atlas + Bitmap Font** 方案：

1. **预处理阶段**：FreeType 库将字体文件栅格化为位图，打包进纹理图集
2. **运行时阶段**：通过 UV 坐标从图集中采样，生成文本网格
3. **渲染阶段**：GPU 执行纹理采样和 Alpha 混合，完成最终显示

该设计在性能（减少 Draw Call）和内存（字形复用）之间取得了良好平衡，适合游戏 UI、2D 渲染等场景。
