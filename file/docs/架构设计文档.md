## 1. 概述

Boo Engine 的 Vulkan 渲染系统是一个**功能完善**、**架构清晰**的渲染封装，提供了从底层 Vulkan API 到上层渲染能力的完整抽象层。

### 🎯 核心特性

| 特性 | 说明 |
|------|------|
| 🎨 双渲染器支持 | Default（默认渲染器）+ Builtin（内置渲染器） |
| 📦 多资源管理 | Shader、Texture、Mesh、Material、RenderTexture |
| 📱 多平台支持 | Window + Android |

## 2. 渲染流程

 Boo Engine 采用**分层渲染架构**，整个渲染流程如下：

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              渲染流程                                           │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   ┌──────────────┐       ┌──────────────┐       ┌──────────────┐              │
│   │   场景数据    │  ──▶  │ Builtin Renderer │  ──▶  │ RenderTexture │            │
│   │  (UI/3D)    │       │  (离屏渲染)    │       │  (离屏纹理)   │            │
│   └──────────────┘       └──────────────┘       └──────┬───────┘              │
│                                                         │                      │
│                                                         ▼                      │
│   ┌──────────────┐       ┌──────────────┐       ┌──────────────┐              │
│   │   交换链      │  ◀──  │ Default Renderer │  ◀──  │ RenderTexture │            │
│   │  (上屏)      │       │  (上屏渲染)    │       │  (渲染目标)   │            │
│   └──────────────┘       └──────────────┘       └──────────────┘              │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 🔄 详细流程

| 步骤 | 渲染器 | 说明 |
|------|--------|------|
| 1️⃣ | - | 收集场景数据（UI 组件、3D 对象等） |
| 2️⃣ | **Builtin Renderer** | 将渲染物渲染到 **RenderTexture**（离屏纹理） |
| 3️⃣ | **Default Renderer** | 将 RenderTexture 作为纹理，通过交换链渲染到屏幕 |

### 💡 核心设计理念

- **Builtin Renderer**：负责将游戏对象渲染到离屏纹理（RenderTexture）
- **Default Renderer**：负责将离屏纹理渲染到屏幕（最终显示）
- 两者配合实现**延迟渲染**或**后处理**效果

## 3. 双渲染器架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              GfxManager (渲染系统管理器)                          │
│                         唯一对外入口，协调所有渲染模块                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                            GfxRenderer (渲染器抽象)                          │
│                    统一接口，管理 Default 和 Builtin                          │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┴───────────────┐
                    ▼                               ▼
        ┌───────────────────────┐       ┌───────────────────────┐
        │   Default Renderer    │       │   Builtin Renderer    │
        │    (默认渲染器)       │       │   (内置渲染器)        │
        │                       │       │                       │
        │ • 上屏渲染            │       │ • 离屏渲染            │
        │ • 渲染目标为交换链    │       │ • 渲染目标为纹理      │
        │ • 显示最终画面        │       │ • 收集渲染数据        │
        │ • 后处理效果         │       │ • 批处理优化          │
        └───────────────────────┘       └───────────────────────┘
```

### 📊 双渲染器对比

| 特性 | Default Renderer | Builtin Renderer |
|------|-----------------|------------------|
| **定位** | 上屏渲染器 | 离屏渲染器 |
| **渲染目标** | 交换链（屏幕） | RenderTexture（离屏纹理） |
| **用途** | 最终画面显示 | 收集场景渲染数据 |
| **流程** | 从纹理读取，渲染到屏幕 | 收集场景数据，渲染到纹理 |

## 4. 默认渲染器 (Default Renderer)

### 4.1 定位与功能

**Default Renderer** 是 Boo Engine 的**上屏渲染器**，负责将渲染结果最终显示到屏幕上。

### 4.2 核心职责

| 功能 | 说明 |
|------|------|
| 🎯 **交换链管理** | 管理 Vulkan 交换链，处理显示同步 |
| 🖼️ **纹理渲染** | 从 RenderTexture 读取像素数据 |
| 🔄 **呈现** | 将渲染结果提交到显示设备 |

### 4.3 渲染流程

```
初始化 → 创建交换链 → 创建渲染通道 → 创建渲染管线 →
每帧: 绑定纹理 → 执行绘制 → 呈现到屏幕
```

---

## 5. 内置渲染器 (Builtin Renderer)

### 5.1 定位与功能

**Builtin Renderer** 是 Boo Engine 的**离屏渲染器**，负责收集场景数据并渲染到离屏纹理。

### 5.2 核心职责

| 功能 | 说明 |
|------|------|
| 📦 **数据收集** | 收集场景中的渲染对象（UI、3D） |
| 🎨 **离屏渲染** | 渲染到 RenderTexture（离屏纹理） |
| ⚡ **批处理** | 自动合批减少 Draw Call |
| 🔀 **多队列** | 支持多个渲染队列 |

### 5.3 渲染管线类型

Builtin Renderer 内置多种渲染管线：

```
GfxBuiltinPipeline (基类)
    │
    ├── GfxBuiltinPipelineUI      // UI 渲染管线
    │       └── 特点：无深度测试、混合开启
    │
    ├── GfxBuiltinPipeline3D      // 3D 渲染管线
    │       └── 特点：有深度测试、无混合
    │
    └── GfxBuiltinPipeline        // 默认管线
```

## 7. Shader 示例

### 7.1 UI Vertex Shader

文件位置：`file/shader/ui/ui.vert`

```glsl
#version 450
#extension GL_ARB_separate_shader_objects : enable

// Uniform Buffer Object
layout(binding = 0) uniform UniformBufferObject {
    mat4 view;      // 视图矩阵
    mat4 proj;      // 投影矩阵
    float time;     // 时间
} ubo;

// 顶点输入属性
layout (location = 0) in vec3 inPos;       // 位置
layout (location = 1) in vec2 inTexCoord;   // 纹理坐标

// 实例输入属性
layout (location = 2) in mat4 inWorldMatrix; // 世界矩阵
layout (location = 6) in vec4 inColor;       // 顶点颜色

// 输出
layout (location = 0) out vec2 uv;
layout (location = 1) out vec4 color;

void main() {
    // 计算世界坐标（Z轴设为0，2D渲染）
    vec4 worldPosition = inWorldMatrix * vec4(inPos.xy, 0.0, 1.0);

    // 视图变换
    vec4 viewPos = ubo.view * worldPosition;

    // 投影变换到裁剪空间
    vec4 clipPos = ubo.proj * viewPos;
    gl_Position = clipPos;

    // Y轴翻转（Vulkan NDC vs OpenGL）
    gl_Position.y = -gl_Position.y;

    // 传递到片段着色器
    uv = inTexCoord;
    color = inColor.rgba;
}
```

### 7.2 UI Fragment Shader

文件位置：`file/shader/ui/ui.frag`

```glsl
#version 450
#extension GL_ARB_separate_shader_objects : enable

// 纹理采样器
layout(binding = 0) uniform sampler2D texSampler;

// 输入
layout (location = 0) in vec2 uv;
layout (location = 1) in vec4 color;

// 输出
layout (location = 0) out vec4 outColor;

void main() {
    // 采样纹理颜色
    vec4 texColor = texture(texSampler, uv);

    // 混合顶点颜色和纹理颜色
    outColor = texColor * color;
}
```

## 9. 总结

### ✅ 架构优点

1. **分层渲染** - 离屏渲染 + 上屏渲染，灵活支持后处理
2. **双渲染器策略** - 职责分离，Builtin 收集数据，Default 显示画面
3. **统一入口** - GfxManager 提供所有渲染操作
4. **模块化清晰** - 内部实现与对外接口分离
5. **Vulkan 封装完善** - 隐藏底层复杂性
6. **支持多平台** - Window + Android

---

> 📝 **后记**：Boo Engine 的渲染系统采用**离屏 + 上屏**的分层架构，Builtin Renderer 负责将场景渲染到离屏纹理，Default Renderer 负责将离屏纹理渲染到屏幕。这种设计使得后处理效果变得更加简单，同时也为多渲染目标提供了良好的支持。
